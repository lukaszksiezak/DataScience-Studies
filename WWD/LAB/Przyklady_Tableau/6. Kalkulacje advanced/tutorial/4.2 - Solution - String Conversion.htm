Concept: Since each Google Maps URL has the same general format, getting the Latitude and Longitude from each URL is just a matter of a recognizing the pattern, then parsing accordingly. For Latitude, we need to get everything from the @ to the comma. For Longitude, we need to get everything from the comma after the @ to the following comma.

 

1.    First, let’s parse out Latitude. Navigate to Analysis and then Create Calculated Field. Name the calculation Position of "@" and type in the following to find where the @ is in the URL:

FIND([Google Maps URL],"@")

 

FIND takes two inputs: a string to search, and a smaller substring to find within the first string. Find returns the position of the substring within the larger string. In this case, we’re looking for the “@” within each URL. In the next calculation, we’ll use this to trim off everything before the “@”. See the screenshot below to see the Position of “@” calculation at work.

 

 

2.    Trim the URL up until the @ with this calculation. Navigate to Analysis and then Create Calculated Field. Name the calculation Trim to "@" and type in the following for the calculation:

MID([Google Maps URL], [Position of "@"]+1)

 

MID takes two (and optionally a third) inputs: a string you want to parse, the starting position of the part of the string you want to keep, (and optionally the length of the part of the string you want to keep -- without this, MID will just keep everything after the starting position). Since we just want to keep everything after “@”, we tell the MID function to cut the URL directly after “@”, indicated by [Position of “@”]+1 in the calculation. See the screenshot below for how this would look in Tableau.

 

 

3.    Navigate to Analysis and then Create Calculated Field. Name the calculation Latitude and enter the following to keep the left-most part of the string, and get rid of the following comma and everything afterward:

FLOAT(LEFT([Trim to "@"],FIND([Trim to "@"],",")-1))

 

This is the last step in parsing out the latitude. Up to this point, we’ve trimmed off everything to the left of the latitude, and now we want to trim off everything to the right of the latitude. In each URL, the end of the latitude is indicated by a comma. Using the FIND function, we search for the first comma we find in our Trim to “@” field and return the position of the comma. Then using the LEFT function, we keep everything to the left of the comma and cut off the rest. Lastly, we wrap the whole string in FLOAT to change its type from a string to a decimal (i.e. a float). This will be important when in the next step when we assign a Geographic Role to this Latitude field. See the screenshot below to see how our new Latitude calculation is working.

 

 

4.    Next, let’s parse out Longitude. Navigate to Analysis and then Create Calculated Field. Name the calculation Position of "," and type in the following to find where the comma is in the URL:

FIND([Google Maps URL],",")

 

FIND takes two inputs: a string to search, and a smaller substring to find within the first string. Find returns the position of the substring within the larger string. In this case, we’re looking for the comma within each URL. In the next calculation, we’ll use this to trim off everything before the comma. See the screenshot below to see the Position of “,” calculation at work.

 

 

5.    Trim the URL up to the comma with this calculation. Navigate to Analysis and then Create Calculated Field. Name the calculation Trim to "," and type in the following for the calculation:

MID([Google Maps URL], [Position of ","]+1)

 

MID takes two (and optionally a third) inputs: a string you want to parse, the starting position of the part of the string you want to keep, (and optionally the length of the part of the string you want to keep -- without this, MID will just keep everything after the starting position). Since we just want to keep everything after the comma, we tell the MID function to cut the URL directly after comma, indicated by [Position of “,”]+1 in the calculation. See the screenshot below for how this would look in Tableau.

 

 

6.    Navigate to Analysis and then Create Calculated Field. Name the calculation Longitude and enter the following to keep the left-most part of the string, and get rid of the following comma and everything afterward:

FLOAT(LEFT([Trim to ","],FIND([Trim to ","],",")-1))

 

This is the last step in parsing out the longitude. Up to this point, we’ve trimmed off everything to the left of the longitude, and now we want to trim off everything to the right of the longitude. In each URL, the end of the longitude is indicated by another comma. Using the FIND function, we search for the first comma we find in our Trim to “,” field and return the position of the comma. (Recall that we already removed the first comma, because we added 1 in our Trim to “,” calculation.) Then using the LEFT function, we keep everything to the left of the comma that indicates the end of the longitude, and cut off the rest. Lastly, we wrap the whole string in FLOAT to change its type from a string to a decimal (i.e. a float). This will be important when in the next step when we assign a Geographic Role to this Longitude field. See the screenshot below to see how our new Longitude calculation is working.

 

 

7.    Now that we have both Latitude and Longitude in their own fields, assign a geographic role to each so we can use them in a map. Right-click or hit the drop-down on the Latitude field in the Dimensions pane, hover over Geographic Role, and select Latitude. Then right-click or hit the drop-down on the Longitude field in the Dimensions pane, hover over Geographic Role, and select Longitude.

 

 

8.    Lastly, let’s make the map. Drag both Latitude and Longitude from the Measures pane into the Dimensions pane, since we’ll want to use these as dimensions. Double-click Latitude or place it on the Rows shelf, and double-click Longitude or place it on the Columns shelf. Drag Park Name to Label on the Marks card.
