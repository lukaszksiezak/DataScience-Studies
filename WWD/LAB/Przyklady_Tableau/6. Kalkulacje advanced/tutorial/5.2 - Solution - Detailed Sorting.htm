View Challenge

 

Concept: The challenge is getting multiple rows of data into a single row.  To do this, we need to use a table calculation to take the previous genre and line them up.  The function PREVIOUS_VALUE can be used to iteratively take the previous value and line it up with the current value.

 

1.    Navigate to the Analysis menu and Create Calculated Field. Name the calculation List with Comma and enter the following formula:

IF INDEX()=1

THEN ATTR([Genre])

ELSE

PREVIOUS_VALUE(ATTR([Genre])) + “, “ + ATTR([Genre])

END

 

 

This calculation is first checking whether the index is 1. This will be true if the genre is the first genre listed. In the picture below, since Drama is the first genre listed under 20th Century Fox, the genre Drama is returned.

 

 

If the index of a genre is not equal to 1, then the calculation is referring to any other row, and will take the previous value, which would be the genre from the previous row, separated by a comma, followed by the current genre. If we bring our List with Comma calculation to Text on the Marks card, our view will look like this:

 

 

This is be computing using Table (Down) which refers to the all of the studios in the table. Instead, we want a list of genres for each studio, so we want to change how it is being computed.

 

2.    Bring List with Comma to the Marks card, and then right-click the List with Comma pill. Select Compute Using and then Pane (Down).

 

 

3.    Bring out Lead Studio and Genre to Rows.

4.    Drag the List with Comma pill from the Marks card to the Text shelf.

 

 

5.    Right-click on Genre and deselect Show Header.

 

 

6.    Next, we need to filter out all the rows besides a single row for each Lead Studio. Navigate to the Analysis menu and Create Calculated Field titled Filter, with the following formula:

RANK([List with Comma])

 

 

This rank is going to look at the string of values in our List with Comma field. This means that whichever pane has the longest string will be ranked the highest. That is, the row that has every genre included will have the highest rank, allowing us to use this later on to filter.

 

7.    Right-click on the Filter field and Convert to Discrete.

8.    Bring this out on the Rows shelf, in between Lead Studio and Genre.

 

 

9.    Right-click on Filter in the Rows shelf and navigate to Compute Using > Pane (Down).

 

Now the RANK calculation will restart based on the pane, which, in this case, is the Lead Studio. Now we can see that the row within the pane that has the longest string is ranked as number one.

 

 

10. Drag this Filter pill onto the Filters shelf and only select 1.

 

 

This selection allows us to filter to only show us the line if the rank is number one. This returns us a single comma separated line for each studio in our data set.

 

11. Lastly, bring the dimension Color onto the Color shelf, so you can clearly see which studios produce movies of more than one genre.

 

We have included the ‘Color’ calculation because it is an LOD expression, meant to simply show the end user if the studio produces more than one genre. Feel free to take a look under the hood, or as a bonus try to recreate it on your own.
