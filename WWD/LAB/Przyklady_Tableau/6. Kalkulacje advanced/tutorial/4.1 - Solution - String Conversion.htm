Concept: Since each Original Measure Name has the same general format, getting the city from each Original Measure Name is just a matter of a few string calculations that build off of each other: chop off the part before the city, chop off the part after the city, and if there’s anything extra hanging around (like a state abbreviation), chop that off too.

 

1.    Write a calculation to remove “Weekly” from each Original Measure Name. Navigate to the Analysis menu, and select Create Calculated Field. Name the calculation “Weekly” Removed and enter the following into the body of the calculation:

MID([Original Measure Names], 8)

 

We want to keep everything to the right of “Weekly” and we know that “Weekly ” (including the space afterwards) is 8 characters long. MID takes two (and optionally a third) inputs: a string you want to parse, the starting position of the part of the string you want to keep, (and optionally the length of the part of the string you want to keep -- without this, MID will just keep everything after the starting position). Since we just want to remove “Weekly” and the following space to start out, we tell the MID function to start keeping the Original Measure Names directly after “Weekly” and the following space. See the screenshot below for how this would look in Tableau.

 

 

2.    Now remove everything from “Regular” onward. Select Analysis and then Create Calculated Field, and name this calculation City (and state). Type the following for your calculation:

LEFT(["Weekly" Removed], FIND(["Weekly" Removed], " Regular")-1)

 

We want to keep everything from the far left until just before the word “Regular.” LEFT takes two inputs: a string you want to parse, and the number of characters you want to keep starting from the left. Since we want to parse the “Weekly” Removed string just before “ Regular” (including the space preceding it), we use “Weekly” Removed for the first input, and use FIND in the second input to get the position of Regular. FIND takes two inputs: a string in which you want to find the position of a smaller piece of string, and the smaller piece of string you want to find. In this case, we’re finding the position of “ Regular” (including the space before).

 

But why the -1? Let’s say we have the string “Tableau” and just want to keep everything before the “e.” That is, we want to keep the first four characters: “Tabl.” We could use this knowledge to write LEFT([“Tableau”, 4), but let’s use the same technique we use above. The “e” is in position five -- that’s exactly one more character than we want to keep. This is always the case when we’re keeping everything before a particular character:

 

 

As such, the -1 in the calculation will always give us the exact length of the string we want to keep. See how the calculation City (and state) is working in the image below.

 

 

3.    Lastly, we need to remove the state abbreviation if there is one. Select Analysis and then Create Calculated Field, and name this calculation City. Type the following for your calculation:

IF CONTAINS([City (and state)], ",")

THEN LEFT([City (and state)], FIND([City (and state)], ",")-1)

ELSE [City (and state)]

END

 

Let’s break down this logic statement. The IF statement checks to see whether there’s a state in our City (and state) string by checking for a comma -- only the strings that have a state will contain a comma. This uses the CONTAINS function, which takes two inputs: a string you want to check, and a smaller substring to check for. The following THEN statement does another string calculation if there is indeed a comma (and therefore a state) in the City (and state) string. As in step 2, this uses the LEFT function to keep just the leftmost characters for the city, and finds where to cut the string using the FIND function. In the case that the City (and state) string does not contain a comma (and therefore does not contain a state), the ELSE statement returns the original City (and state) string.

 

 

And now we’ve gone from a long string to just our city!
